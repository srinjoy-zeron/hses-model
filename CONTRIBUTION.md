# Contributing Guidelines

Thank you for considering contributing to this repository. Please follow the guidelines below to ensure a smooth collaboration.


## How to Contribute

1. Fork the repository.
2. Clone the forked repository in your local machine.
    ```
    git clone "https://github.com/<your-username>/hses-zeron.git
    ```
3. Create a new branch from `main`.
   ```
    git checkout -b feature/your-feature-name
    ````
    OR
    ```
    git checkout -b bug/bug-name
    ``` 
4. Set up the codebase locally using [README.md](README.md)
5. Make your changes with clear, concise commits.
6. Ensure your code follows the projectâ€™s coding standards.
7. Test your changes locally.
8. Add tests and comments for the changes if necessary.
9. Push your branch to your fork.
10. Open a Pull Request (PR) against the `main` branch.

---

## Commit Message Format

* Use clear and descriptive messages.
* Follow this format:

  ```
  type(scope): short description
  ```

  **Examples:**

  * `fix(api): handle null user`
  * `feat(ui): add dark mode`
  * `bug(ui): double-click of calculate button`

---

## Pull Request Guidelines

* Describe **what** you changed and **why**.
* Reference related issues if applicable.
* Keep PRs focused; avoid unrelated changes.
* Ensure all checks pass before requesting review.

---

## Code Style

* Follow existing code patterns and conventions.
* Run linters/formatters if configured.
* Avoid unnecessary refactors.

---

## Reporting Issues

When opening an issue, include:

* Clear title and description describing the issue.
* Steps to reproduce (if applicable) along with any required specifications of system(eg. Operating System, Browser etc.).
* Expected vs actual behavior along with Screenshots or logs if possible.

---

## Security Issues

Do **not** open public issues for security vulnerabilities.
Report them privately to the maintainers.
